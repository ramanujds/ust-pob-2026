## Java Naming Conventions

When writing Java code, it's important to follow standard naming conventions to ensure code readability and maintainability. Here are some common conventions:

1. **Classes and Interfaces**: Use PascalCase (also known as UpperCamelCase) for class and interface names. Each word should start with a capital letter.
   - Example: `MyClass`, `DataProcessor`, `UserService`
   - Example Interface: `Runnable`, `Serializable`
   - Example Abstract Class: `AbstractShape`, `BaseController`
   - Example Enum: `DayOfWeek`, `ColorType`
   - Example Annotation: `Override`, `Deprecated`
   - Example Exception Class: `IOException`, `NullPointerException`

2. **Methods**: Use camelCase for method names. The first word should start with a lowercase letter, and each subsequent word should start with a capital letter.
   - Example: `calculateTotal()`, `getUserName()`, `processData()`
   - Example Getter Method: `getAge()`, `isActive()`
   - Example Setter Method: `setAge()`, `setActive()`
   - Example Boolean Method: `isAvailable()`, `hasPermission()`

3. **Variables**: Use camelCase for variable names, similar to method names.
   - Example: `totalAmount`, `userName`, `dataList`
 
4. **Packages**: Use all lowercase letters for package names. Separate words with dots.
   - Example: `com.example.myapp`, `org.project.module`
   - Example Subpackage: `com.example.myapp.utils`, `org.project.module.services`

5. **Constants**: Use all uppercase letters with words separated by underscores for constant variable names.
   - Example: `MAX_VALUE`, `DEFAULT_TIMEOUT`, `PI`
   - Example Static Final Variable: `public static final int MAX_USERS = 100;`
